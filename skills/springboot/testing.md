# Spring Boot Testing — AI Agent Context

This file defines the authoritative patterns for writing tests in Spring Boot applications.
Agents MUST follow these patterns when generating or modifying test code.

---

## Core Principles

- Follow the testing pyramid: Unit (70%), Integration (20%), E2E (10%).
- Tests must be deterministic, isolated, and fast.
- Mock external dependencies in unit tests; use real infrastructure in integration tests via Testcontainers.
- Target 80%+ coverage for service and controller layers. Never chase 100% at the expense of test quality.
- NEVER test trivial getters, setters, or constructors generated by Lombok.
- Tests are first-class code: apply the same quality standards as production code.

---

## Test Naming Convention

Use the pattern: `methodName_givenCondition_expectedBehavior`

```java
// Examples
createOrder_givenValidRequest_returnsCreatedOrder()
createOrder_givenInsufficientStock_throwsOutOfStockException()
findById_givenNonExistentId_throwsEntityNotFoundException()
processPayment_givenExpiredCard_returnsPaymentFailureResult()
```

DO:
- Use snake_case to separate the three parts of the name.
- Make the condition and expectation precise.

DON'T:
- NEVER use vague names like `testCreate()`, `test1()`, or `shouldWork()`.

## Unit Tests — Service Layer

```java
@ExtendWith(MockitoExtension.class)
class OrderServiceTest {

    @Mock
    private OrderRepository orderRepository;

    @Mock
    private ProductRepository productRepository;

    @Mock
    private PaymentService paymentService;

    @InjectMocks
    private OrderService orderService;

    @Captor
    private ArgumentCaptor<Order> orderCaptor;

    private Product testProduct;
    private User testUser;

    @BeforeEach
    void setUp() {
        testProduct = ProductFixture.aProduct()
            .withId(UUID.randomUUID())
            .withStock(10)
            .withPrice(BigDecimal.valueOf(99.99))
            .build();

        testUser = UserFixture.aUser()
            .withId(UUID.randomUUID())
            .build();
    }

    @Test
    void createOrder_givenValidRequest_returnsCreatedOrder() {
        // Arrange
        CreateOrderRequest request = new CreateOrderRequest(testProduct.getId(), 2);
        Order expectedOrder = Order.builder()
            .userId(testUser.getId())
            .productId(testProduct.getId())
            .quantity(2)
            .status(OrderStatus.PENDING)
            .build();

        given(productRepository.findById(testProduct.getId()))
            .willReturn(Optional.of(testProduct));
        given(orderRepository.save(any(Order.class)))
            .willReturn(expectedOrder);

        // Act
        Order result = orderService.createOrder(request, testUser.getId());

        // Assert
        assertThat(result).isNotNull();
        assertThat(result.getStatus()).isEqualTo(OrderStatus.PENDING);
        assertThat(result.getQuantity()).isEqualTo(2);

        verify(orderRepository).save(orderCaptor.capture());
        Order capturedOrder = orderCaptor.getValue();
        assertThat(capturedOrder.getUserId()).isEqualTo(testUser.getId());
        assertThat(capturedOrder.getProductId()).isEqualTo(testProduct.getId());
    }

    @Test
    void createOrder_givenInsufficientStock_throwsOutOfStockException() {
        // Arrange
        CreateOrderRequest request = new CreateOrderRequest(testProduct.getId(), 99);
        given(productRepository.findById(testProduct.getId()))
            .willReturn(Optional.of(testProduct)); // stock is 10

        // Act & Assert
        assertThatThrownBy(() -> orderService.createOrder(request, testUser.getId()))
            .isInstanceOf(OutOfStockException.class)
            .hasMessageContaining(testProduct.getId().toString());

        verify(orderRepository, never()).save(any());
    }

    @Test
    void findById_givenNonExistentId_throwsEntityNotFoundException() {
        UUID unknownId = UUID.randomUUID();
        given(orderRepository.findById(unknownId)).willReturn(Optional.empty());

        assertThatThrownBy(() -> orderService.findById(unknownId))
            .isInstanceOf(EntityNotFoundException.class);
    }

    @AfterEach
    void tearDown() {
        // Cleanup if needed (usually not required with MockitoExtension)
    }
}
```

DO:
- Use `@ExtendWith(MockitoExtension.class)` — do NOT use `@SpringBootTest` for unit tests.
- Use `@Mock` for dependencies, `@InjectMocks` for the class under test.
- Use `@Captor` to capture and verify arguments passed to mocks.
- Use BDD-style `given/willReturn` from `org.mockito.BDDMockito` for readability.
- Follow the Arrange-Act-Assert (AAA) pattern with blank lines separating each section.
- Use `assertThatThrownBy` for exception assertions.

DON'T:
- NEVER use `@SpringBootTest` for pure service logic tests — it loads the full context.
- NEVER use `assertTrue(result != null)` — use AssertJ's `assertThat(result).isNotNull()`.
- NEVER use `assertEquals` from JUnit — always use AssertJ.
- NEVER assert on `e.getMessage()` using `contains` with fragile strings — check the exception type first.

## Unit Tests — Controller Layer (@WebMvcTest)

```java
@WebMvcTest(OrderController.class)
@Import(SecurityConfig.class) // only if security filter needs to be active
class OrderControllerTest {

    @Autowired
    private MockMvc mockMvc;

    @MockBean
    private OrderService orderService;

    @MockBean
    private UserDetailsServiceImpl userDetailsService; // required by security config

    @Autowired
    private ObjectMapper objectMapper;

    @Test
    @WithMockUser(roles = "USER")
    void createOrder_givenValidRequest_returns201() throws Exception {
        CreateOrderRequest request = new CreateOrderRequest(UUID.randomUUID(), 2);
        OrderResponse response = OrderResponse.builder()
            .id(UUID.randomUUID())
            .status("PENDING")
            .build();

        given(orderService.createOrder(any(CreateOrderRequest.class), any(UUID.class)))
            .willReturn(response);

        mockMvc.perform(post("/api/v1/orders")
                .contentType(MediaType.APPLICATION_JSON)
                .content(objectMapper.writeValueAsString(request)))
            .andExpect(status().isCreated())
            .andExpect(jsonPath("$.id").isNotEmpty())
            .andExpect(jsonPath("$.status").value("PENDING"));
    }

    @Test
    @WithMockUser(roles = "USER")
    void getOrder_givenNonExistentId_returns404() throws Exception {
        UUID unknownId = UUID.randomUUID();
        given(orderService.findById(unknownId))
            .willThrow(new EntityNotFoundException("Order not found: " + unknownId));

        mockMvc.perform(get("/api/v1/orders/{id}", unknownId))
            .andExpect(status().isNotFound());
    }

    @Test
    void createOrder_givenUnauthenticated_returns401() throws Exception {
        CreateOrderRequest request = new CreateOrderRequest(UUID.randomUUID(), 2);

        mockMvc.perform(post("/api/v1/orders")
                .contentType(MediaType.APPLICATION_JSON)
                .content(objectMapper.writeValueAsString(request)))
            .andExpect(status().isUnauthorized());
    }

    @Test
    @WithMockUser(roles = "USER")
    void createOrder_givenInvalidRequest_returns400() throws Exception {
        CreateOrderRequest invalidRequest = new CreateOrderRequest(null, -1); // null productId

        mockMvc.perform(post("/api/v1/orders")
                .contentType(MediaType.APPLICATION_JSON)
                .content(objectMapper.writeValueAsString(invalidRequest)))
            .andExpect(status().isBadRequest())
            .andExpect(jsonPath("$.errors").isArray());
    }
}
```

DO:
- Use `@WebMvcTest(XxxController.class)` to load only the web layer.
- Use `@MockBean` for all service dependencies.
- Use `@WithMockUser` to simulate an authenticated user.
- Test all HTTP status codes: 200, 201, 400, 401, 403, 404, 500.
- Use `jsonPath` assertions for response body verification.

DON'T:
- NEVER use `@SpringBootTest` for controller slice tests — it is too slow and broad.
- NEVER test controller logic that belongs in the service layer.

## Integration Tests (@SpringBootTest)

```java
@SpringBootTest(webEnvironment = SpringBootTest.WebEnvironment.RANDOM_PORT)
@AutoConfigureMockMvc
@Testcontainers
@TestPropertySource(properties = {
    "spring.jpa.hibernate.ddl-auto=validate",
    "application.security.jwt.expiration=3600000"
})
class OrderIntegrationTest {

    @Container
    static PostgreSQLContainer<?> postgres = new PostgreSQLContainer<>("postgres:16-alpine")
        .withDatabaseName("testdb")
        .withUsername("test")
        .withPassword("test");

    @DynamicPropertySource
    static void configureProperties(DynamicPropertyRegistry registry) {
        registry.add("spring.datasource.url", postgres::getJdbcUrl);
        registry.add("spring.datasource.username", postgres::getUsername);
        registry.add("spring.datasource.password", postgres::getPassword);
    }

    @Autowired
    private MockMvc mockMvc;

    @Autowired
    private ObjectMapper objectMapper;

    @Autowired
    private UserRepository userRepository;

    @Autowired
    private JwtService jwtService;

    private String userToken;

    @BeforeEach
    void setUp() {
        User user = userRepository.save(UserFixture.aUser().build());
        userToken = jwtService.generateToken(user);
    }

    @AfterEach
    void tearDown() {
        userRepository.deleteAll();
    }

    @Test
    void createOrder_givenValidRequest_persistsAndReturns201() throws Exception {
        CreateOrderRequest request = new CreateOrderRequest(/* ... */);

        mockMvc.perform(post("/api/v1/orders")
                .header("Authorization", "Bearer " + userToken)
                .contentType(MediaType.APPLICATION_JSON)
                .content(objectMapper.writeValueAsString(request)))
            .andExpect(status().isCreated())
            .andExpect(jsonPath("$.id").isNotEmpty());
    }
}
```

DO:
- Use `@Testcontainers` with a `PostgreSQLContainer` for realistic database tests.
- Use `@DynamicPropertySource` to wire container properties into Spring.
- Use `WebEnvironment.RANDOM_PORT` to avoid port conflicts.
- Use `@TestPropertySource` to override production properties for test scenarios.
- Clean up test data in `@AfterEach` to keep tests independent.

DON'T:
- NEVER use H2 for integration tests against production PostgreSQL behavior (use Testcontainers).
- NEVER share state between integration tests.

## Repository Tests (@DataJpaTest)

```java
@DataJpaTest
@AutoConfigureTestDatabase(replace = AutoConfigureTestDatabase.Replace.NONE)
@Testcontainers
class OrderRepositoryTest {

    @Container
    static PostgreSQLContainer<?> postgres = new PostgreSQLContainer<>("postgres:16-alpine");

    @DynamicPropertySource
    static void configureProperties(DynamicPropertyRegistry registry) {
        registry.add("spring.datasource.url", postgres::getJdbcUrl);
        registry.add("spring.datasource.username", postgres::getUsername);
        registry.add("spring.datasource.password", postgres::getPassword);
    }

    @Autowired
    private OrderRepository orderRepository;

    @Autowired
    private TestEntityManager entityManager;

    @Test
    @Transactional
    void findByUserId_givenExistingOrders_returnsUserOrders() {
        UUID userId = UUID.randomUUID();
        Order order1 = entityManager.persistAndFlush(OrderFixture.anOrder().withUserId(userId).build());
        Order order2 = entityManager.persistAndFlush(OrderFixture.anOrder().withUserId(userId).build());

        List<Order> results = orderRepository.findByUserId(userId);

        assertThat(results).hasSize(2)
            .extracting(Order::getId)
            .containsExactlyInAnyOrder(order1.getId(), order2.getId());
    }

    @Test
    @Transactional
    void findByStatus_givenNoMatchingOrders_returnsEmptyList() {
        List<Order> results = orderRepository.findByStatus(OrderStatus.COMPLETED);
        assertThat(results).isEmpty();
    }
}
```

DO:
- Use `@DataJpaTest` for repository-only slice tests.
- Use `@AutoConfigureTestDatabase(replace = NONE)` to disable H2 replacement when using Testcontainers.
- Use `TestEntityManager` for setting up test data within the JPA context.
- Annotate test methods with `@Transactional` to roll back after each test.

## Test Data Builders / Fixtures Pattern

```java
// Fixture builder pattern — use builder DSL to construct test objects
public class OrderFixture {

    public static OrderFixture.Builder anOrder() {
        return new Builder();
    }

    public static class Builder {
        private UUID id = UUID.randomUUID();
        private UUID userId = UUID.randomUUID();
        private OrderStatus status = OrderStatus.PENDING;
        private BigDecimal total = BigDecimal.valueOf(100.00);
        private LocalDateTime createdAt = LocalDateTime.now();

        public Builder withId(UUID id) { this.id = id; return this; }
        public Builder withUserId(UUID userId) { this.userId = userId; return this; }
        public Builder withStatus(OrderStatus status) { this.status = status; return this; }
        public Builder withTotal(BigDecimal total) { this.total = total; return this; }

        public Order build() {
            return Order.builder()
                .id(id)
                .userId(userId)
                .status(status)
                .total(total)
                .createdAt(createdAt)
                .build();
        }
    }
}
```

DO:
- Create a fixture class per domain entity in a `fixtures` or `support` test package.
- Provide sensible defaults for all required fields.
- Expose `withXxx(...)` methods for overriding only the fields relevant to the test.

DON'T:
- NEVER inline complex object construction in test methods — extract to fixtures.
- NEVER use `new Order()` with no-args constructor and set fields via setters in tests.

## AssertJ Assertions — Common Patterns

```java
// Basic equality
assertThat(result.getId()).isEqualTo(expected.getId());
assertThat(result.getStatus()).isEqualTo(OrderStatus.PENDING);

// Nullability
assertThat(result).isNotNull();
assertThat(result.getDeletedAt()).isNull();

// Collections
assertThat(orders).hasSize(3);
assertThat(orders).isEmpty();
assertThat(orders).isNotEmpty();
assertThat(orders).containsExactlyInAnyOrder(order1, order2);
assertThat(orders).extracting(Order::getStatus).containsOnly(OrderStatus.PENDING);
assertThat(orders).anySatisfy(o -> assertThat(o.getId()).isEqualTo(targetId));

// Exceptions
assertThatThrownBy(() -> service.findById(unknownId))
    .isInstanceOf(EntityNotFoundException.class)
    .hasMessageContaining("not found");

// Numbers / dates
assertThat(result.getTotal()).isGreaterThan(BigDecimal.ZERO);
assertThat(result.getCreatedAt()).isBefore(LocalDateTime.now());

// Strings
assertThat(result.getEmail()).startsWith("user@");
assertThat(result.getName()).isNotBlank();
```

DO:
- Always use AssertJ. Import `assertThat` from `org.assertj.core.api.Assertions`.
- Use `extracting` to assert on properties of collection elements.
- Use `satisfies` or `anySatisfy` for complex element assertions.

DON'T:
- NEVER use JUnit's `assertEquals`, `assertTrue`, `assertNotNull` — always use AssertJ.
- NEVER chain assertions without using AssertJ's fluent API.

## Security Testing

```java
// @WithMockUser — injects a mock user with specified roles
@Test
@WithMockUser(username = "user@example.com", roles = {"USER"})
void getProfile_givenAuthenticatedUser_returns200() throws Exception {
    mockMvc.perform(get("/api/v1/profile"))
        .andExpect(status().isOk());
}

// @WithMockUser with ADMIN role
@Test
@WithMockUser(roles = "ADMIN")
void deleteUser_givenAdminRole_returns204() throws Exception {
    mockMvc.perform(delete("/api/v1/admin/users/{id}", UUID.randomUUID()))
        .andExpect(status().isNoContent());
}

// Test forbidden access
@Test
@WithMockUser(roles = "USER")
void deleteUser_givenUserRole_returns403() throws Exception {
    mockMvc.perform(delete("/api/v1/admin/users/{id}", UUID.randomUUID()))
        .andExpect(status().isForbidden());
}

// @WithUserDetails — loads a real UserDetails from UserDetailsService
@Test
@WithUserDetails("admin@example.com")
void adminEndpoint_givenRealAdminUser_returns200() throws Exception {
    mockMvc.perform(get("/api/v1/admin/dashboard"))
        .andExpect(status().isOk());
}
```

DO:
- Always test all three authentication scenarios: unauthenticated (401), forbidden (403), authorized (200).
- Use `@WithMockUser` for simple role-based tests.
- Use `@WithUserDetails` when the actual `UserDetails` instance matters (e.g., for `@AuthenticationPrincipal`).

## Transactional Test Rollback

```java
@DataJpaTest
@Transactional  // rolls back after each test method
class ProductRepositoryTest {

    @Autowired
    private ProductRepository productRepository;

    @Test
    void save_givenValidProduct_persistsWithId() {
        Product product = ProductFixture.aProduct().build();
        Product saved = productRepository.save(product);
        assertThat(saved.getId()).isNotNull();
    }
    // Database is rolled back here — next test starts clean
}
```

DO:
- Annotate `@DataJpaTest` test classes with `@Transactional` for automatic rollback.
- Use `@Rollback(false)` only when you need to inspect committed state (rare).

DON'T:
- NEVER annotate `@SpringBootTest` web layer tests with `@Transactional` — it does not roll back HTTP-triggered transactions.

## Testcontainers Setup (shared container)

```java
// Base class to share the container across multiple integration test classes
@Testcontainers
public abstract class BaseIntegrationTest {

    @Container
    static final PostgreSQLContainer<?> POSTGRES =
        new PostgreSQLContainer<>("postgres:16-alpine")
            .withReuse(true); // requires ~/.testcontainers.properties: testcontainers.reuse.enable=true

    @DynamicPropertySource
    static void registerProperties(DynamicPropertyRegistry registry) {
        registry.add("spring.datasource.url", POSTGRES::getJdbcUrl);
        registry.add("spring.datasource.username", POSTGRES::getUsername);
        registry.add("spring.datasource.password", POSTGRES::getPassword);
    }
}

// Extend in tests
class OrderIntegrationTest extends BaseIntegrationTest { ... }
class UserIntegrationTest extends BaseIntegrationTest { ... }
```

## Summary Checklist

- [ ] Test naming: `methodName_givenCondition_expectedBehavior`
- [ ] Unit tests use `@ExtendWith(MockitoExtension.class)`, not `@SpringBootTest`
- [ ] Controller tests use `@WebMvcTest` + `MockMvc` + `@MockBean`
- [ ] Integration tests use `@SpringBootTest` + `@Testcontainers`
- [ ] Repository tests use `@DataJpaTest` + Testcontainers
- [ ] Test data managed via fixture builders
- [ ] All assertions use AssertJ (no JUnit assertions)
- [ ] Security tests cover 401, 403, and 200 scenarios
- [ ] Exception tests use `assertThatThrownBy`
- [ ] Transactional rollback on `@DataJpaTest` tests
- [ ] No tests for trivial Lombok-generated methods
- [ ] 80%+ coverage target for service and controller layers
